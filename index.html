Welcome to the Gromox project webpage

Gromox is the central groupware server component of grommunio. It is capable of serving as a replacement for Microsoft Exchange and compatibles. Connectivity options include RPC/HTTP (Outlook Anywhere), MAPI/HTTP, IMAP, POP3, an SMTP-speaking LDA, and a PHP module with a Z-MAPI function subset.

Gromox relies on other components to provide a sensibly complete mail system, such as Postfix as a mail transfer agent, and grommunio-admin for user management. A webmail client interface is available with grommunio-web. The grommunio Appliance ships these essentials and has a ready-to-run installation of Gromox.

Support is available through grommunio GmbH and its partners. See https://grommunio.com/ for details.

The source code repository and technical issue tracker can be found at https://github.com/grommunio/gromox.

  • 2022-01-09. Version 1.12 was released. kdb2mt's splice mode was improved to detect the mapping for Inbox, and the documentation about MRO field caveats was updated. pff2mt has received updates and scans more properties and recipients now.   • 2021-12-07. Version 1.10 was released. Cached Mode now works with plaintext-only messages and messages whose recipients are incomplete. Expanded notes on GH.
  • 2021-11-27. Version 1.9 was released. The kdb2mt importer's -s option now recognizes public stores. Incoming mail now has the Content-Disposition header respected with regards to inline/hidden attachments. Modifying embedded messages now properly saves the changes. Expanded notes on GH.
  • 2021-11-13. Version 1.8 was released. A few fixes in vCard parsing, folder cleaning and offline mode were made. Expanded notes on GH.
  • 2021-11-07. Version 1.7 was released. It improves importing from Zarafa/Kopano. Delivery to private shared stores is now recognized. Expanded notes on GH.
  • 2021-10-31. Version 1.6 was released. This fixes a number of issues with gromox-delivery-queue/gromox-delivery where the older versions may generate invalid temporary files and then crash on it. An Out Of Office autorespond mechanism was added. The kdb2mt/mt2exm importer has seen updates. Expanded notes on GH.
  • 2021-10-08. Version 1.4 was released. This cures an Outlook crash in conjunction with OP_MOVE-type Inbox Rules. pff2mt now skips bogus MAPI properties and continues rather than exiting. Expanded notes on GH.
  • 2021-09-01. Version 1.2 was released. This includes a much improved gromox-kdb2mt (importer for Kopano database/mailboxes) which fixes bugs in the named property resolution for appointment/contact data, supports embedded message objects, and supports reading gzip attachments. Expanded notes on GH.
  • 2021-08-06. Version 1.0 was released. This comes with a reworked PFF command line interface as well as an importer for Kopano mailboxes.
  • 2021-07-13. Version 0.27 was released. This has a store owner mode, rectifies some issues with calendar objects, supports "working elsewhere" busy status, and repairs umlaut garbage in mail previews (new mails). Expanded notes on GH.
  • 2021-06-09. Version 0.25 was released. Since 0.23, a number of logic fixes have been made, such as resolving garbled IMAP folder names, and skipping bogus properties in PST file imports. Expanded notes on GH.
  • 2021-05-13. Version 0.23 was released. Since 0.20, there have been a number of crash fixes, support for send quotas, a new utility "gromox-pffimport" for PFF/PST/OST importing, and a change to the default mysql_adaptor upgrade check.
